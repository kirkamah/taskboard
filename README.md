# Taskboard — инструкция по запуску

Это настоящий сайт с авторизацией, личными досками, комнатами и ролями. Всё подключено к Supabase.

## Что тебе нужно

1. Аккаунт **Supabase** (у тебя уже есть) — для базы и авторизации
2. Аккаунт **GitHub** — для хранения кода
3. Аккаунт **Vercel** — для размещения сайта в интернете

Все три — бесплатные.

---

## Шаг 1. Второй SQL-скрипт в Supabase (2 минуты)

В папке проекта есть файл `supabase_02_join_room.sql`. Его нужно запустить в Supabase — он добавляет функцию для присоединения к комнате по коду.

1. Зайди в свой проект в Supabase
2. Слева — иконка **SQL Editor** (`>_`)
3. Создай новый запрос (New query)
4. Открой файл `supabase_02_join_room.sql`, скопируй весь текст, вставь в окно
5. Нажми **Run**
6. Должно быть «Success. No rows returned»

## Шаг 2. Загрузить код на GitHub (10 минут)

### 2.1 Создать аккаунт на GitHub
- Зайди на https://github.com/signup
- Зарегистрируйся (email, пароль, ник)
- Подтверди email

### 2.2 Создать новый репозиторий
- Нажми **+** в правом верхнем углу → **New repository**
- Имя: `taskboard` (или любое другое)
- Видимость: **Private** (приватный — код не будет публичным)
- **НЕ** ставь галочки «Add README», «Add .gitignore», «Add license» — оставь пустой репозиторий
- Нажми **Create repository**

### 2.3 Загрузить файлы через браузер
- На странице пустого репозитория увидишь «uploading an existing file» — нажми эту ссылку
- Либо: вкладка **Code** → кнопка **Add file** → **Upload files**
- Перетащи **все файлы и папки проекта** (кроме `node_modules`, если вдруг есть) в окно загрузки
  - Обязательно должны быть: папки `app`, `components`, `lib`, файлы `package.json`, `next.config.mjs`, `tailwind.config.js`, `postcss.config.mjs`, `jsconfig.json`, `middleware.js`, `.gitignore`
  - **НЕ загружай** `.env` (если он есть) — это секретные ключи, они пойдут в Vercel напрямую
- Внизу: **Commit changes** → **Commit directly to main branch** → **Commit changes**
- Подожди, пока файлы загрузятся (пара минут)

## Шаг 3. Задеплоить на Vercel (5 минут)

### 3.1 Регистрация
- Зайди на https://vercel.com/signup
- Нажми **Continue with GitHub** (это привяжет твой GitHub к Vercel — быстрее всего)
- Разреши доступ

### 3.2 Импорт проекта
- На главной Vercel нажми **Add New** → **Project**
- Увидишь список своих GitHub-репозиториев. Найди `taskboard`
- Нажми **Import**

### 3.3 Настройка переменных окружения (САМОЕ ВАЖНОЕ!)
- Откроется страница конфигурации
- Найди раздел **Environment Variables**
- Добавь две переменные:

| Name | Value |
|------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://ybvfyferfvebxxeqyhvm.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `sb_publishable_jlzKWpHIL01VSTMyKuaFeA_5PfyRiSv` |

(Это те самые ключи, что ты получал из Supabase. Если будут отличаться — используй свежие из Supabase → API Keys.)

- Нажми **Deploy**
- Подожди 2-3 минуты

### 3.4 Готово!
- Получишь ссылку вида `https://taskboard-xxxx.vercel.app`
- Открой её — увидишь свой сайт с регистрацией/входом!

## Шаг 4. Добавить URL сайта в Supabase (2 минуты)

Чтобы авторизация корректно работала с настоящего домена, а не только с localhost:

1. Зайди в Supabase → **Authentication** → **URL Configuration**
2. В поле **Site URL** впиши свой Vercel-URL: `https://taskboard-xxxx.vercel.app`
3. В поле **Redirect URLs** добавь `https://taskboard-xxxx.vercel.app/**` (со звёздочками)
4. Нажми **Save**

## Всё! Что дальше

Теперь можно:
- Регистрироваться на своём сайте по email
- Создавать личные задачи
- Создавать комнаты и делиться кодами
- Приглашать друзей — пусть тоже регистрируются и присоединяются по коду

**Тестирование:** проще всего открыть сайт в двух разных браузерах (Chrome и Firefox) или в обычном + инкогнито — зарегистрировать двух пользователей и проверить, как они взаимодействуют в одной комнате.

## Как потом обновлять сайт

Когда я пришлю новый код (например, с дизайном):
1. Удали старые файлы в своём GitHub-репозитории и загрузи новые (через Upload files)
2. Vercel **автоматически** пересоберёт сайт и опубликует новую версию за 2-3 минуты
3. Данные в базе останутся на месте — ничего не потеряется

## Если что-то пошло не так
